- **YC Companies** (https://www.ycombinator.com/companies) — who's already building here

- **Reforgers** (https://reforgers.com/) — YC ideas worth rebuilding
- **StartupIdeasDB** (https://startupideasdb.com/) — validated ideas
- **NeedGap** (https://needgap.com/) — real unsolved problems
- **Product Hunt** (https://www.producthunt.com/) — what's trending now
- **Exploding Topics** (https://explodingtopics.com/) — what's about to trend

Summarize key findings before ideating.

## Phase 1: Diverge (generate)
Generate 10+ ideas using these lenses:
- **What's broken?** — friction people tolerate but shouldn't
- **What failed before but timing is right now?** — reference startups.rip/loot-drop findings
- **What's adjacent?** — natural extension of Qash/Prism/my skills
- **What does YC want?** — cross-reference with YC RFS
- **What's contrarian?** — goes against current consensus but has a thesis
- **What's combinatorial?** — merge two unrelated domains (e.g., ZK + fintech + AI)

For each idea, note the inspiration source (which website, company, or failed startup sparked it).

## Phase 2: Filter (evaluate)
Score each idea on:
- Feasibility (can I build an MVP in 2 weeks?)
- Market pull (are people already searching for this?)
- Founder-market fit (does my blockchain/ZK/AI background give me an edge?)
- Monetization clarity (is there an obvious way to charge?)
- Timing (why now and not 2 years ago or 2 years from now?)
- Synergy with Qash/Prism (does this strengthen my existing projects?)
